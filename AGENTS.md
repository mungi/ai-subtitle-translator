# AGENTS.md

이 저장소에서 작업할 때의 프로젝트별 협업 규칙입니다.

## 작업 원칙

- 사용자가 계획 검토를 요청한 단계에서는 기능 개발을 시작하지 않는다.
- 변경은 작게 나누고, 각 단계마다 검증 방법을 함께 남긴다.
- Udemy와 YouTube를 우선 대상으로 한다.
- 문서는 한국어를 기준 원본으로 작성한다. 영어와 일본어 문서는 한국어 문서를 번역해 내용과 구조를 동기화한다.

## 소개 웹사이트 작업

- `mungi.github.io/ai-subtitle-translator`에 표시되는 소개 웹사이트는 이 저장소의 `site/` 디렉터리에서 관리한다.
- GitHub Pages 배포 워크플로는 `site/`만 artifact로 업로드한다. 확장 프로그램 코드와 테스트는 Pages artifact에 포함하지 않는다.
- 소개 웹사이트 작업 전후에는 이 저장소에서 `git status`와 원격 저장소 설정을 확인하고, 변경 후 한국어·영어·일본어 표시와 언어 선택 동작을 검증한다.
- 확장 프로그램과 소개 웹사이트를 함께 변경해야 하면, 각각의 검증 결과를 커밋 또는 PR 설명에 남긴다.

## 품질 기준

- 자막 데이터는 `{ id, start, end, text }` 형태의 내부 표준 모델로 먼저 정규화한다.
- 번역 엔진은 UI, 자막 수집, 자막 렌더링과 분리한다.
- LLM 번역은 전체 자막 맥락을 활용하되, 결과는 원래 자막 cue 시간에 맞춰 다시 배치해야 한다.
- API 키는 `chrome.storage.local`에 저장하는 MVP로 시작하고, 문서에 보안 한계를 명시한다.
- 네트워크 호출은 가능하면 background/service worker 계층으로 모은다.

## 검증 기준

- YouTube: 자막 목록 조회, 자막 cue 수집, 원문 표시, 번역 표시.
- Udemy: 강의 플레이어 감지, 자막 목록 조회, 자막 cue 수집, 원문 표시, 번역 표시.
- LLM 번역: 원본 cue 개수와 번역 cue 개수가 일치하고, 시간 범위가 보존된다.
- 실패 시: 번역 실패 메시지를 보여주고 원문 자막 표시를 유지한다.
